# CS-230-Portfolio-submission-8-2

The Gaming Room wanted to expand Draw It or Lose It from Android to a scalable, web-based application that supports multiple users and platforms. The system needed to handle real-time gameplay while remaining responsive and reliable.

In this project, I focused on connecting design decisions directly to system requirements. I recommended a cloud-based Linux environment for scalability, used a client-server architecture with REST APIs for cross-platform communication, and designed memory usage around performance by loading images on demand instead of storing all ~1.6 GB in memory. Storage decisions included using a database for user data and cloud/file storage for large image assets.

The design document helped structure the development process by breaking the system into architecture, memory, storage, and communication. This made it easier to understand how components interact before implementation.

If I were to improve the document, I would refine earlier sections to match the clarity and detail of later ones. As I progressed, my explanations became more precise and better aligned with the application.

Overall, this project reinforced the importance of designing with user needs in mind—especially performance, scalability, and usability—while building systems that can operate across multiple platforms.
